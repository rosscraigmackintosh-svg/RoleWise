# Rejection logging
Rejection is data, not judgement.
## Flow

Rejection → Logged → Pattern (if repeated)

## Behaviour
- record what happened
- capture reason if available
- avoid over-analysis
## Product goal
Preserve signal without damaging confidence.
## In one sentence
Log the outcome, don't over-learn from it.
