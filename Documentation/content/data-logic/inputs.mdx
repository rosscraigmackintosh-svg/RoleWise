# Inputs
Rolewise works from a limited and explicit set of inputs.
## Primary input
The core input is the job description text provided by the user.
Everything starts here.
## Supporting inputs
Additional inputs may include:
- recruiter emails (pasted manually)
- user decisions (apply, skip, watch)
- stage updates (interview, rejection, offer)
- notes and context added by the user
## Input discipline
Rolewise does not:
- scrape personal data
- track behaviour invisibly
- rely on hidden signals

Everything should be visible and user-provided.
## Why this matters
Clear inputs create clear boundaries.
If the system is only as good as what it sees, the user should understand what it is seeing.
## In one sentence
Rolewise works from explicit, user-provided inputs — not hidden data.
