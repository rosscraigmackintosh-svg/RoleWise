# Design system
The Rolewise design system exists to support clarity.
It is not decorative. It is the set of rules that keeps the product readable, consistent, and calm as it grows.
## Purpose
Rolewise deals with interpretation, trade-offs, and uncertainty.
The interface should help the user think clearly, not compete for attention.
## What it covers
The design system defines:
- visual principles
- typography
- colour usage
- spacing and layout
- component behaviour
- states and labels
- content structure
## What it should feel like
The product should feel:
- calm
- structured
- readable
- deliberate
- quiet but confident
## What it should avoid
Avoid:
- dashboard noise
- bright status-heavy UI
- gamification
- unnecessary animation
- decorative clutter
## In one sentence
The design system exists to make Rolewise easy to think inside.
