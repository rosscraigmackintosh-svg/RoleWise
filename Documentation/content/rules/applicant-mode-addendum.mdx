# Applicant Mode addendum
This addendum captures operating rules that sit alongside the main Applicant Mode rules.
These rules handle practical edge cases and product-specific behaviours.
## Hard no rule for production coding
If a role requires production-level coding from the designer, it should be treated as a hard no.
Prototype or exploratory coding is acceptable. Production delivery responsibility is not.
## Verification points
If a role is missing key information such as salary, location, work model, or coding expectations, Applicant Mode should not force a strong conclusion.
These gaps should be surfaced as verification points.
## Salary rule
If salary is stated annually, monthly equivalents should also be shown.
If salary is not stated, it should be marked clearly as not stated.
## Stage tracking rule
Where a role progresses, stage tracking should be consistent with the shared stage model used across Rolewise.
## Risk flag rule
Only capture a limited number of clear structural risk flags. These should be useful for pattern recognition later, not a long list of vague concerns.
## In one sentence
The addendum handles the practical rules Applicant Mode needs to stay useful in real-world role analysis.
