# Outcome interpretation rules
Outcomes need careful interpretation.
A rejection does not always mean a role was a good test of fit. A lack of response does not always mean the user was a poor match.
## Core principle
Rolewise should separate outcome from evaluation quality.
If the hiring process was weak, automated, vague, or structurally noisy, the product should not treat the outcome as strong evidence about the user.
## Low-signal outcomes
Examples of low-signal outcomes include:
- automated rejection with no engagement
- one-way video screening as primary evaluation
- generic recruiter filtering with weak role understanding
- ghosting after application with no meaningful interaction

These should be treated cautiously.
## Higher-signal outcomes
Higher-signal outcomes usually involve:
- thoughtful human interaction
- discussion of role scope or judgement
- deeper process stages
- role-specific feedback or clear engagement

These outcomes can carry more weight.
## Reflection rule
Rolewise should help the user learn from repeated patterns, but not over-learn from single weak outcomes.
## In one sentence
Outcome interpretation rules protect the user from drawing strong conclusions from weak hiring signals.
