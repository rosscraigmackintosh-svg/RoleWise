# How to use this manual

This manual is a living reference document. It is intended for the team building and maintaining Rolewise — not as user-facing documentation.

## What is in here

The manual covers the following areas:

- **Overview** — What Rolewise is, what it is not, and how to read this document
- **Philosophy** — The thinking and values that underpin every product decision
- **Constitution** — The non-negotiable rules that govern how the product works
- **Bible** — Vision, positioning, and long-term direction
- **Product** — How each feature works, in detail
- **Rules** — Specific operational rules governing features and data interpretation
- **Design System** — Visual and tone standards
- **Data & Logic** — What goes in, what comes out, and how
- **Workflows** — Step-by-step process documentation
- **Templates** — Reusable formats for recurring tasks
- **Changelog** — A record of documentation and product decisions

## How to navigate it

Use the left sidebar to move between sections. The navigation reflects the structure of the product and its documentation.

Pages are ordered sequentially but work as standalone references.

## How to update it

All content lives in MDX files inside the `content/` folder. To update a page, edit the corresponding `.mdx` file. No code changes are required. See the engineering notes for the full update process.

## Conventions used

Throughout this manual:

- **Bold text** marks important terms or concepts on first use
- `code style` is used for field names, values, and technical references
- Blockquotes mark principles or rules that have special weight

> This format is used for rules, doctrines, or constraints that should not be overridden without deliberate decision.
